# 结果条工具栏布局优化设计
日期：2026-03-11

## 背景

当前结果条顶部工具栏在窄宽度下会出现两个问题：

1. `点击 / 自动` 切换控件显示后，与覆盖按钮区域发生视觉重叠。
2. 语言按钮 `AUTO ▾`、`简中 ▾` 在当前宽度下容易显示不全。

用户确认的目标行为：

1. 顶部工具栏继续保持单行，并通过横向滚动容纳内容。
2. `点击 / 自动` 切换控件在显示时，放到“覆盖”按钮右侧。
3. 语言按钮需要保证 `AUTO ▾` 和 `简中 ▾` 之类的文本能完整显示。

## 设计决策

### 布局策略

继续使用现有的“左侧滚动工具栏 + 右侧固定图标按钮”结构，不改成两行。

滚动工具栏内部顺序调整为：

- 模式按钮
- 源语言按钮
- 目标语言按钮
- 复制译文按钮
- 覆盖按钮
- `点击 / 自动` 切换控件

这样切换控件会稳定出现在覆盖按钮右侧，不再插在覆盖按钮左边。

### 尺寸刷新

问题根因是滚动区内容宽度只在初始化时计算一次，后续：

- 切换控件显隐变化
- 语言按钮文本变化

都不会重新刷新内部内容宽度，导致滚动区仍按旧宽度排布，视觉上出现挤压与重叠。

修复方式：

- 将滚动容器 widget 和布局保存为实例属性。
- 新增一个专门的工具栏刷新方法，统一在以下时机调用：
  - 初始化完成后
  - 模式切换后
  - 源语言文本更新后
  - 目标语言文本更新后

刷新方法负责：

- 重新计算语言按钮最小宽度
- `adjustSize()`
- 更新滚动内容 widget 的最小宽度/推荐宽度

### 语言按钮宽度

为源语言和目标语言按钮设置基于字体测量的最小宽度，确保至少能完整显示：

- `AUTO ▾`
- `简中 ▾`
- 以及菜单切换后常见短标签

不会把按钮宽度写死成过大的常量，而是按文本宽度加左右 padding 计算，避免不同 DPI 下失真。

## 代码边界

### `src/ui/result_bar.py`

变更内容：

- 调整滚动工具栏内按钮顺序
- 保存滚动工具栏内部 widget / layout 引用
- 新增工具栏刷新方法
- 在模式切换、语言文本变化时触发刷新
- 为语言按钮设置合理最小宽度

不改动：

- 控制器逻辑
- 托盘逻辑
- 设置存储逻辑

## 测试要点

1. 固定/多框模式下，切换控件在覆盖按钮右侧。
2. 切换控件显示后，工具栏内容宽度会重新增长。
3. 源语言和目标语言按钮最小宽度足以容纳当前文本。
