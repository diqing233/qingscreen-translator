# 关闭按钮行为与托盘恢复设计

日期：2026-03-11

## 背景

当前主结果条右上角 `✕` 直接调用 `hide()`，行为过于隐式：

- 用户点击后不知道程序是退出了，还是仍在后台运行
- 现有系统托盘只能承载入口，不能明确恢复主结果条
- 关闭行为不能在设置中调整

用户确认的目标行为如下：

1. 第一次点击 `✕` 时弹出选择框，让用户决定“放到托盘”还是“退出程序”
2. 对话框提供“记住我的选择，下次不再询问”
3. 对话框明确提示：之后可以去“设置 -> 通用”修改
4. 设置页增加“关闭按钮行为”选项，可改为“每次询问 / 最小化到托盘 / 直接退出程序”
5. 如果选择放到托盘，用户可以通过托盘图标再次打开主结果条

---

## 设计决策

### 方案选择

采用“控制器统一处理关闭策略”的方案：

- `ResultBar` 只负责发出“用户点击关闭”的信号
- `CoreController` 统一决定是询问、隐藏到托盘还是退出程序
- `SystemTray` 只负责托盘入口和恢复主窗口
- `SettingsStore / SettingsWindow` 负责行为持久化和可配置化

不把关闭逻辑直接塞进 `ResultBar`，原因是该逻辑属于应用级生命周期，而不是单个控件自身职责。

### 设置模型

新增设置项：

- `close_button_behavior`

可选值：

- `ask`：每次询问
- `tray`：最小化到托盘
- `quit`：直接退出程序

默认值：

- `ask`

说明：

- “记住我的选择”不单独存布尔值
- 当用户在首次弹窗中勾选“记住我的选择”并选择动作时，直接把 `close_button_behavior` 更新为 `tray` 或 `quit`
- 若用户取消勾选，则仅执行本次动作，不改设置

---

## 关闭交互

### 首次或询问模式

当 `close_button_behavior == 'ask'` 时，点击 `✕` 打开确认对话框。

对话框内容：

- 标题：`关闭程序`
- 主文案：`关闭后要做什么？`
- 辅助文案：`之后可以在“设置 -> 通用 -> 关闭按钮行为”里修改`
- 按钮：`放到托盘`、`退出程序`、`取消`
- 复选框：`记住我的选择，下次不再询问`

默认交互：

- 复选框默认勾选
- 推荐主路径为“放到托盘”

行为规则：

- 选“放到托盘” + 勾选记住
  - 本次隐藏到托盘
  - 保存设置为 `tray`
- 选“退出程序” + 勾选记住
  - 本次直接退出
  - 保存设置为 `quit`
- 未勾选记住
  - 只执行本次动作
  - 设置保持 `ask`
- 点“取消”
  - 什么都不做

### 直接托盘模式

当 `close_button_behavior == 'tray'` 时：

- 点击 `✕` 不再弹窗
- 主结果条隐藏
- 程序继续常驻托盘

### 直接退出模式

当 `close_button_behavior == 'quit'` 时：

- 点击 `✕` 不再弹窗
- 直接调用应用退出

---

## 托盘恢复行为

现有托盘能力保留，并补齐主结果条恢复入口。

新增行为：

- 托盘菜单增加 `显示主窗口`
- 点击托盘图标时，恢复主结果条

恢复动作定义：

- 显示 `ResultBar`
- 执行 `raise_()`
- 执行 `activateWindow()`

本次不强制联动其他窗口：

- 不自动恢复设置窗口
- 不自动恢复历史窗口
- 不改变现有固定翻译框/多框的显隐状态

这样可以避免引入额外状态同步问题。

---

## 后台状态约束

当主结果条已经隐藏到托盘后：

- 后台翻译结果只更新内部 UI 状态
- AI 科普结果只更新内部 UI 状态
- 错误提示只更新内部 UI 状态
- 不主动弹出主结果条

只有以下操作才恢复主结果条：

- 用户点击托盘图标
- 用户点击托盘菜单的 `显示主窗口`

---

## 托盘不可用时的降级

若系统托盘不可用：

- `tray` 路径不能进入“隐藏后无法恢复”的状态
- 控制器应提示用户当前系统托盘不可用
- 然后执行直接退出

这样可以避免程序表面消失、但用户无法找回主窗口。

## 代码边界

### `src/ui/result_bar.py`

变更：

- 新增 `close_requested` 信号
- 右上角 `✕` 从 `self.hide` 改为发出 `close_requested`
- 保留已有的显示、恢复、最小化到任务栏代理能力

不负责：

- 不直接弹关闭确认框
- 不直接决定托盘或退出策略

### `src/core/controller.py`

新增职责：

- 监听 `result_bar.close_requested`
- 根据 `close_button_behavior` 路由动作
- 负责弹出关闭确认框
- 负责隐藏到托盘
- 负责从托盘恢复主结果条
- 负责托盘不可用时的降级

### `src/ui/tray.py`

变更：

- 新增 `show_main_requested` 信号
- 菜单增加 `显示主窗口`
- 托盘图标激活时优先用于恢复主结果条

### `src/core/settings.py`

变更：

- 默认设置新增 `close_button_behavior: 'ask'`

### `src/ui/settings_window.py`

变更：

- 在“通用”页增加 `关闭按钮行为` 选项
- 选项值对应 `ask / tray / quit`
- 允许用户在首次选择之后再次修改

---

## 测试要点

1. `SettingsStore` 默认返回 `close_button_behavior == 'ask'`
2. 设置保存后可正确持久化 `ask / tray / quit`
3. `ask` 模式下选择“放到托盘 + 记住”会保存 `tray`
4. `ask` 模式下选择“退出程序 + 记住”会保存 `quit`
5. `ask` 模式下未勾选“记住”时，不修改设置
6. `tray` 模式下点击 `✕` 会隐藏主结果条，不退出程序
7. `quit` 模式下点击 `✕` 会调用 `app.quit()`
8. 托盘菜单存在 `显示主窗口`
9. 点击托盘图标后可以恢复主结果条
10. 托盘不可用时，`tray` 路径会提示并降级退出
